If you are building an **agent with OpenAI models (o4 → GPT-5) and using DSPy**, the good news is that **DSPy is designed to make model upgrades easy**. You usually **do not rewrite prompts manually**; instead you change the **LM configuration**, and DSPy will optimize prompts automatically.

Below is the typical workflow.

---

# 1. Install DSPy

First install the library.

```bash
pip install dspy-ai
```

---

# 2. Configure the OpenAI Model in DSPy

DSPy uses an **LM object** to define the model.

Example with an older model:

```python
import dspy

lm = dspy.LM(
    model="openai/o4-mini",
    api_key="YOUR_API_KEY"
)

dspy.configure(lm=lm)
```

To upgrade to a new model (like GPT-5), you only change the model name.

```python
lm = dspy.LM(
    model="openai/gpt-5",
    api_key="YOUR_API_KEY"
)

dspy.configure(lm=lm)
```

No prompt rewrite needed.

---

# 3. Define a DSPy Signature (Instead of Writing Prompts)

DSPy replaces prompts with **structured signatures**.

Example:

```python
class AnswerQuestion(dspy.Signature):
    """Answer a question using context."""
    
    context = dspy.InputField()
    question = dspy.InputField()
    
    answer = dspy.OutputField()
```

This becomes the **prompt automatically generated by DSPy**.

---

# 4. Build a Module

Example RAG-style agent module.

```python
class QA(dspy.Module):
    
    def __init__(self):
        self.predict = dspy.Predict(AnswerQuestion)

    def forward(self, context, question):
        return self.predict(context=context, question=question)
```

Usage:

```python
qa = QA()

result = qa(
    context="The capital of France is Paris.",
    question="What is the capital of France?"
)

print(result.answer)
```

---

# 5. Let DSPy Optimize the Prompt Automatically

Instead of manually tuning prompts, DSPy uses **optimizers**.

Example using **BootstrapFewShot**.

```python
optimizer = dspy.BootstrapFewShot()

optimized_program = optimizer.compile(
    QA(),
    trainset=train_data
)
```

DSPy will automatically:

* generate better prompts
* select few-shot examples
* adapt prompts for the model

So when switching from **o4 → GPT-5**, the optimizer can **re-tune the prompt**.

---

# 6. Using DSPy with Tools (Agent Style)

Example tool integration.

```python
class SearchTool(dspy.Signature):
    query = dspy.InputField()
    results = dspy.OutputField()
```

Agent module:

```python
class Agent(dspy.Module):

    def __init__(self):
        self.search = dspy.Predict(SearchTool)
        self.answer = dspy.Predict(AnswerQuestion)

    def forward(self, question):
        docs = self.search(query=question)
        return self.answer(context=docs.results, question=question)
```

---

# 7. Switching Models Safely

When switching models:

1. change LM configuration
2. re-run DSPy optimizer
3. evaluate results

Example:

```python
lm = dspy.LM("openai/gpt-5")
dspy.configure(lm=lm)

optimized_program = optimizer.compile(program, trainset=trainset)
```

---

# 8. Why DSPy Helps with Model Upgrades

Without DSPy:

```text
manual prompt engineering
rewrite prompts
retune few-shot examples
```

With DSPy:

```text
change model
rerun optimizer
prompts adapt automatically
```

This makes it ideal for **rapid LLM upgrades**.

---

✅ **Key takeaway**

When using DSPy:

* prompts are **generated from signatures**
* upgrading models usually means **changing the LM configuration**
* DSPy optimizers **automatically adjust prompts**
